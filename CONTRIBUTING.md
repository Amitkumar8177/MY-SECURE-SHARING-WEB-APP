# Contributing

Thank you for considering contributing to this project! The following guidelines will help your contribution get accepted quickly.

How to contribute

1. Fork the repository and create a feature branch from `main` (or `master`):

```powershell
# example
git checkout -b feature/your-feature
```

2. Make small, focused changes. Include tests where applicable.
3. Run linters and tests (if present) before submitting.
4. Open a pull request with a clear description of the change and why it is needed.

Coding conventions

- Keep code style consistent with the existing project.
- Add comments for complex logic.
- Validate inputs and handle errors gracefully.

Reporting bugs

- Open an issue with steps to reproduce, expected and actual behavior, and any relevant logs.

Security

- Do not include secrets or private data in PRs or issues. If you discover a security vulnerability, contact the repository owner privately.

License

- Contributions will be accepted under the project's license (see `LICENSE`).
